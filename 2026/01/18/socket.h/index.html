POSIX socket 网络编程接口与常用函数详解

  1. 包含linux下网络编程的函数，除send与recv其他函数成功均返回0，失败返回-1（socket成功返回监听套接字fd，accept同理返回已连接套接字fd）

    • int bind (int __fd, __CONST_SOCKADDR_ARG __addr, socklen_t __len) __THROW
    • int connect (int __fd, __CONST_SOCKADDR_ARG __addr, socklen_t __len)
    • ssize_t send (int __fd, const void *__buf, size_t __n, int __flags)
    • ssize_t recv (int __fd, void *__buf, size_t __n, int __flags)
    • int listen (int __fd, int __n) __THROW
    • int accept (int __fd, __SOCKADDR_ARG __addr,socklen_t *__restrict __addr_len)
    • int shutdown (int __fd, int __how) __THROW
    • int setsockopt(int sockfd, int level, int optname, const void *optval, socklen_t optlen)
    • read默认返回收到字节数，若为FIN，则返回零，若为RST，则为-1（阻塞也为-1，与RST的区别是errno设置值不一样）

  3. bind函数的addr参数一般与netinet/in.h的sockaddr_in结构体搭配，即sockaddr与sockaddr_in相互转换
  4. listen第二个参数代表请求队列的最大长度
  5. accept默认为阻塞到有连接请求到达，非阻塞下立即返回-1（若需要非阻塞，则需要在创建套接字时使用SOCK_NONBLOCK，即socket(AF_INET | SOCK_NONBLOCK,SOCK_STREAM,)）
  6. shutdown（立即发送FIN，执行TCP四次分手）的第二个参数指定关闭那方的数据传输：

TCP套接字迭代服务器编程流程

这种情况下，服务器一次只需要处理一个客户端请求，因此基本流程为:

1.   创建socket()，即int fd=socket(AF_INET,SOCK_STREAM,0)生成TCP套接字

2.   使用bind()绑定本机特定端口，即bind(fd,(struct sockaddr*)&server_addr,sizeof(server_addr));

3.   使用listen()使套接字文件描述符进入监听状态，即listen(fd,1)，1表示同时最大监听数

4.   使用死循环，在该循环内使用accept()函数接受连接accept(fd, (struct sockaddr*)&address, (socklen_t*)&addrlen))

5.   处理请求

6.   关闭与当前服务器连接，继续循环使用accept获取下一个等待队列客户端

UDP套接字服务器编程流程

与TCP不一样的地方是不需要使用监听端口，因为UDP是无连接的：

1.   创建socket()，即int fd=socket(AF_INET,SOCK_DGRAM,0)

2.   使用bind()绑定端口

3.   使用死循环，在该循环内使用recvfrom()，即recvfrom(fd,buffer,BUFFER_SIZE,0,(struct sockaddr*)&client_addr,&client_addr_len)，将客户端的数据写入缓冲区buffer，同时将客户端地址信息写入sockaddr结构体

4.   处理请求

5.   不需要关闭连接，循环直接回到recvfrom()等待下一个请求
