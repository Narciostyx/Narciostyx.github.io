C++多线程编程

C++中使用的多线程支持库

  • 通常包含库<thread>，<condition_variable>，<atomic>，<mutex>，<shared_mutex>，<future>

thread

  • 包含一命名空间this_thread
  • 类
    1. thread，单个执行线程，允许多个函数并发执行，操作有join，detach，swap，native_handle（返回底层实现定义的线程句柄），joinable
      • thread必须在销毁前join，否则将引发异常
      • 不支持拷贝，支持移动
    2. jthread，相较于thread，它会在析构时自动加入join，且支持外部请求终止
      • (头文件)使用stop_token类传递给该线程，线程内置的stop_source会自动把其停止令牌给stop_token
      • 内部的停止令牌操作：get_stop_source（返回与线程共享停止状态关联的 stop_source 对象），get_stop_token（返回与线程共享停止状态关联的 stop_token），request_stop（通过线程的共享停止状态请求停止执行）
      • stop_callback，注册回调函数，当stop_token收到请求时会立即执行

mutex

  • 包含类mutex，lock_guard，unique_guard，scope_lock
  • 全局命名空间包含函数try_lock，lock，call_once
  1. mutex包含lock，try_lock，unlock，native_handle
  2. unique_lock包含lock，try_lock（_for或_until），unlock，release（仅解除关联而非释放锁）
  3. scope_lock是互斥体包装器，如：
  1. 递归锁recursive_mutex
  2. 方法call_once，即使被多个线程调用，也只会调用一次
  3. 共享锁shared_mutex

condition_variable

  • condition_variable，通知线程有notify_one，notify_all，使线程等待有wait，wait_for与wait_until

Atomic

  • 原子类型，其操作为原子操作

Note — CAS

  • 即Compare and Swap操作
  • CAS操作需要输入两个数值，一个旧值（期望检查时应该为的值）与一个新值
  • 如果匹配旧值，则替换为新值，反之不替换
  • 方法有load（获取原子对象值），store（保存原子对象值），exchange，wait，notify_one，notify_all，第二个参数均为内存顺序
  • RMW操作（读-修改-写操作）：fetch_add，fetch_sub，fetch_max(\\min\\and\\or\\xor)执行对应操作，再返回之前持有的值，其第二个参数均为内存顺序
  • 内存顺序：
    • 即避免编译优化导致部分内存操作顺序与预想顺序不一致
    • 对于原子操作，其修改操作不可能同时进行，且所有线程均认可这个顺序，即若某个线程看到8先于9，则另外一个相同操作的线程也会看到8先于9
    • happends_before，若操作a在操作b之前发生，则操作a的结果对于操作b可见，但CPU或编译器并不一定会这样执行
    • squenced_before，即单线程中，语句是按顺序执行的
    • synchronizes_with，即不同线程间实现happends_before，如果线程 1 中的操作 a “synchronizes-with” 线程 2 中的操作 b, 则操作 a “inter-thread happens-before” 操作 b，两者均具有传递性
    1. memory_order_relaxed，不做任何同步与排序约束于其他读写，每个线程看到的修改顺序不一致
    2. memory_order_acquire，针对load读操作，即acquire操作

Note

  • 后续读/写操作不会被重排到当前acquire操作之前，若另一个线程对同一个原子变量执行release操作，则两者synchronizes-with，保证release之前的所有发生的写入，在acquier操作之后都对当前线程可见（即传递给acquire操作）
1. `memory_order_release`，针对store写操作，即release操作

Note

  • 是RMW操作，则使用memory_order_acq_rel
  • 若要建立明确的同步关系应该使用release与acquire配对
1. `memory_order_acq_rel`，即2与3的结合
2. `memory_order_seq_cst`，顺序一致性

Future

#future
#async
#promise

  • std::future提供了访问异步操作结果的机制
  • 在标准库中存在两种期望：唯一期望与共享期望
    • 唯一期望，即std::future<>只能与应该指定事件关联
    • 共享期望，即std::shared_future<>能关联多个事件
  • std::future内部保存一个将来会被某个provider赋值的值，因此提供了一个访问该值的方法get()，但如果试图在该方法可用前调用，则会阻塞直到该值可用
    • get()方法只能调用一次，否则抛出异常
  • 有效的std::future对象有以下三种Provider创建：
    • std::async
    • std::promise::get_future
    • std::packaged_task::get_future

Note
<future>头文件包含以下类与函数：

  • provider类：std::promise与std::package_task
  • provider函数：std::async
  • 其他类型：std::future_error，std::future_errc，std::future_status，std::launch
  • std::future的拷贝操作是被禁止的
    • valid成员函数查看是否拥有共享状态，默认构造函数为false，移动后一般为true
    • get成员函数
    • wait成员函数，等待共享标志为ready，在此之前一直阻塞
    • wait_for，即前者带超时机制
    • wait_until，与前者逻辑一致
    • share成员函数，返回一个shared_future

Note — shared_future
与std::future类似，但允许拷贝，以及允许多次get方法调用

Note — 返回值
wait_for与wait_until的返回值为future_status，其标志位有ready,timeout,deferred

  • std::promise保存某一类型T的值，该值可被future对象获取，因此其提供了一种线程同步手段，其构造时可与一个共享状态（一般std::future）相关联
    • 可以使用get_future获取与之相关联的future对象
    • 禁止拷贝

Note
