C++命名修饰(C++ name mangling)

简要总结了MSVC与Itanium C++ ABI的修饰规则

  • 主体由GCC与Clang遵循的Itanium C++ ABI以及MSVC的私有修饰规则

GCC

  • 以_Z开头
  • 函数名称长度与函数名称，如3add
  • 函数参数类型，如ii，即int,int
  • 默认情况下不考虑返回值类型，大多数情况下重载决议是通过参数类型列表，所在命名空间以及限定符进行
  • int add(int a,int b)：_Z3addii
  • 命名空间（类同理）以N开头，E结尾，如namespace myspace：N7myspaceE，namespace myspace{ void func(int a,int b); }：_ZN7myspace4funcEii
  • 模板以I开头，E结尾，如template<typename T> void func(T arg){...}，则实例化模板func<int>(int)：_Z4funcIiEi
  • external "C"则是强制使用C语言命名修饰，方便被C语言程序调用
  • 针对构造与析构函数有特殊编码：
  1. C1: 完全构造函数（用于直接构造对象）
  2. C2: 基类构造函数（用于派生类构造函数中调用基类部分）
  3. C3: 别名构造函数（用于new表达式，构造后返回指针）
  4. D0: 删除析构函数（用于delete表达式，调用析构并释放内存）
  5. D1: 完全析构函数（用于直接析构对象）
  6. D2: 基类析构函数（用于派生类析构函数中调用基类部分）

MSVC

  • 以?开头
  • 函数名称
  • 复合编码，在MSVC中，函数命名修饰除了作用域，参数列表，还包含返回类型，调用约定，顺序为函数类型（普通函数，成员函数，其中成员函数根据其访问修饰符也不一样），调用约定、返回类型、参数列表，Z代表结束（若无参数则直接为Z，反之有参数则为@Z），以@@开始，如void __cdecl func(){...}：?func@@YAXXZ，void __stdcall bar(int a,float b){...}：?bar@@YGXHM@Z
  • 命名空间，以类似@namespace的方式，如namespace myspace{ void __cdecl func(); }：?func@myspace@@YAXXZ，**嵌套命名空间（从内向外）**则为@Inner@Outer
