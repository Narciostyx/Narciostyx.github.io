C++ MYSQL编程

MySQL C API 接口与常用函数的使用方法

  • 包含MYSQL结构体以及相关函数
  • MYSQL结构体，代表一个客户端到MYSQL服务器的连接
  • MYSQL_RES，代表MYSQL数据库查询结果的返回集合

函数

  • mysql_init(MYSQL*)初始化一个MYSQL结构体，在后续操作中被用作与MySql服务器的连接句柄，如果传入的是NULL，则会分配一个新的MYSQL结构并返回其指针
  • mysql_real_connect(xxx)用于建立与MySql服务器的实际连接，参数为：指向MYSQL结构体的指针，主机名，用户名，密码，数据库，端口，如果运行在本地并通过Unix套接字连接则为套接字的路径（若为TCP连接，则为NULL），客户端标志
  • mysql_close(MYSQL*)关闭连接
  • mysql_query(MYSQL*,const char*)用于发送SQL查询，现被mysql_real_query或其他预处理语句函数取代

Note
即使查询成功也应该检查mysql_store_result与mysql_use_result的返回值，因为这些函数可能会在尝试检索结果时遇到错误

Note — P.s.
执行select show describe explain必须使用mysql_store_result或mysql_use_result获取结果集

  • mysql_real_query比上面那个查询函数多了字符串长度的参数，因此传入字符串不一定要以\0结尾
  • mysql_store_result、mysql_free_result与mysql_use_result
  • mysql_fetch_row
  • mysql_fetch_lengths
  • mysql_num_fields，获取字段数，与mysql_field_count类似，但是后者传入MYSQL结构体指针
  • mysql_num_rows，获取总行数
  • mysql_real_escape_string(MYSQL*,char*,const char*,unsigned long)，用于转义特殊字符以防止SQL注入，char*是储存转义后字符串的缓冲区

Note

  1. 初始化MYSQL结构体mysql_init
  2. 连接服务器mysql_real_connect
  3. 查询mysql_query等
  4. 返回数据检查，处理mysql_store_result mysql_use_result
  5. 清除结果集mysql_free_result
  6. 关闭连接mysql_close
  • mysql_send_query()，发送查询，不等待结果
  • mysql_next_result()，获取下一个结果集（多语句）
  • 预处理语句
  • 事务控制
